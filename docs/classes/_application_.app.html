state

  • get state(): string

  • Inherited from App.state

    • Defined in node_modules/@loopback/core/dist/application.d.ts:43

    Get the state of the application. The initial state is created and it can transition as follows by start and stop:

    1. start
      • !started -> starting -> started
      • started -> started (no-op)
    2. stop
      • started -> stopping -> stopped
      • !started -> stopped (no-op)

    Two types of states are expected:

    • stable, such as started and stopped
    • in process, such as booting and starting

    Operations such as start and stop can only be called at a stable state. The logic should immediately set the state to a new one indicating work in process, such as starting and stopping.

    Returns string

    • ValueType

    Parameters

    • Optional pattern: string | RegExp | BindingFilter

      A filter function, a regexp or a wildcard pattern with optional * and ?. Find returns such bindings where the key matches the provided pattern.

      For a wildcard:

      • * matches zero or more characters except . and :
      • ? matches exactly one character except . and :

      For a filter function:

      • return false to exclude it.
